System Integrations

Integration Architecture

Webhook-based event-driven architecture. RESTful APIs, service mesh communication, and asynchronous message passing for loosely coupled services.

SIEM to SOAR

Wazuh to TheHive

POST /api/alert/create

Wazuh alerts automatically create cases in TheHive. Alert metadata, severity, and observables extracted for investigation.

SOAR to AUTOMATION

TheHive to Shuffle

POST /api/v1/hooks/{id}

Case updates trigger Shuffle workflows. Automated playbooks execute response actions based on severity.

AI to SIEM

ML Inference to Wazuh

POST /predict

Alert features sent to ML API for real-time prediction. Classification results enriched back into Wazuh alerts.

MONITORING to ALL

Prometheus Scraping

GET /metrics

13 scrape targets across all stacks. Service health, resource utilization, and custom metrics.

ANALYSIS to ENRICHMENT

Alert Triage to RAG

POST /search

Vector search over threat intelligence databases. Contextual enrichment with historical attack patterns.

ALERTING to NOTIFICATION

AlertManager to Slack

POST /services/{webhook}

Critical infrastructure alerts routed to Slack. Email and webhook notifications for service degradation.

Docker Compose Orchestration

Modular deployment across 5 independent compose files. Deploy incrementally or all-at-once for complete AI-SOC infrastructure. 

# DEPLOYMENT SEQUENCE
1. docker compose -f phase1-siem-core-windows.yml up -d
2. docker compose -f phase2-soar-stack.yml up -d
3. docker compose -f monitoring-stack.yml up -d
4. docker compose -f network-analysis-stack.yml up -d  # Linux only
5. docker compose -f ai-services.yml up -d

# HEALTH CHECK
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# VERIFICATION
curl -k https://localhost:443          # Wazuh Dashboard
curl http://localhost:3000             # Grafana
curl http://localhost:8500/health      # ML Inference API
